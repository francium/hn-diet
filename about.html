About

This project was inspired by Paul Houle's Hacker News for Hackers article.

The current filter is based on a blacklist of phrases and domains. Each lines (of the text area on the web page) is used to filter the story titles or domain if prefixed with 'site:'.

Example of a blacklist: 

          site:aws.amazon.com
          site:bbc.com
          site:bloomberg.com
          site:nytimes.com
          site:qz.com
          site:reuters.com
          site:techcrunch.com
          site:theatlantic.com
          site:theverge.com
          site:www.economist.com
          site:www.theguardian.com
          site:www.vox.com
          site:www.washingtonpost.com
          site:zdnet.com
          bitcoin
          housing market
          (IOT.*security)|(security.*IOT)
          apple
          instagram
          iphone
          is hiring
As you can see, it does support regular expressions, so you have some freedom to play around and create somewhat complex patterns to match on.

Privacy

This site uses the Hacker News Firebase API to get the data. This site is served directly from GitHub Pages. You are welcome, and encouraged, to see the source code on github/francium/hn-diet.

No information is stored by this website directly. There maybe information that GitHub or Firebase store as a consequence of using their services -- such as when you request this page from GitHub Pages and when this site loads data using the Firebase Hacker News API. For information stored by Github or Firebase, please review their respective privacy policies.

As an experiment, and to get a sense of the number of people making use of this website, an image will be loaded from the GitHub repo every time the main page of this website is loaded. This information will be stored on the GitHub repo for this website. This is done strictly to get a rough estimate of the popularity of this website only, and I will not have access to any specific information.

Being an experiment, I do not know if it will actually work.

This method of collecting basic analytic data was chosen over services like Google Analytics to better respect user privacy, and not force users to expose their information to an additional party.
